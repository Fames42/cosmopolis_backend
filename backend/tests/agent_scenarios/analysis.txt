# Top 3 issues across all scenarios

1. **Unnecessary friction in scheduling**
   - In several cases, the agent insisted on an exact time before searching, instead of offering available windows based on broader preferences like “tomorrow morning” or “after 18:00.”

2. **Occasional classification/urgency inconsistencies**
   - Examples:
     - Pipe leak possibly under-prioritized.
     - Bathroom door not closing possibly under-prioritized.
     - AC not cooling classified under heating/HVAC ambiguously.
     - FAQ scenario lacked clear FAQ classification behavior.

3. **Template-like phrasing in some Russian turns**
   - Phrases like “Понял(а)” / “Зафиксировал(а)” feel less natural and more bot-like than the rest of the conversations.

   - In most service flows, the agent used tools in the correct order and avoided unnecessary actions.

2. **Strong multilingual performance**
   - Russian and English responses were both fluent, natural, and appropriate to the tenant’s language.

3. **Very good handling of escalation, cancellation, rescheduling, and ticket updates**
   - The agent reliably distinguished between creating, modifying, canceling, and escalating requests.

# Recommendations for improvement

1. **Reduce scheduling friction**
   - Allow broader preference handling:
     - “tomorrow morning”
     - “after 18:00”
     - “first available”
   - Search first, then present matching options instead of forcing exact-time input.

2. **Tighten classification and urgency rules**
   - Define clearer policy for:
     - active leaks
     - privacy/security-related door issues
     - HVAC/cooling vs heating
     - vague complaints that may need escalation if diagnosis remains uncertain

3. **Improve FAQ workflow**
   - If FAQ content is available, answer directly.
   - If not, avoid partial speculative answers like “usually on weekdays.”
   - Explicitly classify FAQ before escalation if that is part of the expected protocol.

4. **Polish conversational naturalness**
   - Replace template-heavy wording with more natural variants:
     - “Понял вас”
     - “Спасибо, записал”
     - “Хорошо, уточню”
   - Avoid gendered placeholder forms like “Понял(а).”

5. **Make provisional vs confirmed scheduling clearer**
   - Especially in reschedule flows, distinguish:
     - available slot found
     - tenant confirmed
     - system reschedule completed
